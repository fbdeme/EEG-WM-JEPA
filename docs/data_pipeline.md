# Data Pipeline Design

> 최종 업데이트: 2026-03-31
> 버전: v1.0

## 핵심 원칙

**"채널 수를 인위적으로 조작하지 않는다."**

Channel Mixer(LUNA 제안, Laya/LuMamba 채택)가 가변 채널을 고정 크기로 압축하는 것이 모델의 핵심 기여이므로, 데이터 파이프라인에서는 채널을 건드리지 않고 있는 그대로 넘긴다.

---

## 사전학습 데이터셋

| 데이터셋 | 규모 | 채널 분포 | 접근성 |
|---|---|---|---|
| **TUH EEG Corpus** | ~1.6TB | 주로 19~22ch | 신청 필요 (무료) |
| **REVE Dataset** | ~4.4TB (92개 혼합) | 82% 저밀도(≤24ch), 3% 중밀도(25~64ch), 15% 고밀도(65~128ch+) | HuggingFace 공개 |

**REVE 실제 채널 분포** (공식 사이트 기반 검증):
- 19ch: 25,176시간 (41.0%) — 주로 TUH
- 21ch: 21,594시간 (35.2%) — 주로 ICARE
- 22ch: 3,673시간 (6.0%)
- 고밀도(125~129ch): ~14.7%

## 전체 파이프라인 아키텍처

```
┌─────────────────────────────────────────────────────┐
│ Offline Preprocessing (1회성)                        │
│                                                     │
│ .edf 원본                                            │
│   ↓                                                 │
│ ① 채널명 표준화 (Alias → 10-20, REVE Position Bank)   │
│ ② 유효 좌표 없는 채널 드롭 (드롭 비율 로깅)              │
│    → 유효 채널 50% 미만인 데이터셋은 제외               │
│ ③ 리샘플링 → 200Hz                                    │
│ ④ Bandpass → 0.5~75Hz                                │
│ ⑤ 2초 윈도우 분할 (400 samples)                        │
│   ↓                                                 │
│ LitData 저장 (샘플 단위 메타데이터 포함)                 │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ Online — Dataset.__getitem__                         │
│                                                     │
│ LitData에서 샘플 로드                                  │
│   ↓                                                 │
│ ⑥ CAR 적용 (패딩 전이므로 마스킹 불필요)                 │
│   x = x - x.mean(dim=0, keepdim=True)               │
│   ↓                                                 │
│ 반환: { eeg: [C,400], coords: [C,3], num_channels: C }│
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ Online — Collate Function                            │
│                                                     │
│ max_c = max(배치 내 각 샘플의 num_channels)            │
│                                                     │
│ ⑦ eeg: [B, max_c, 400] (부족분 zero-padding)         │
│ ⑧ coords: [B, max_c, 3] (부족분 zero-padding)        │
│ ⑨ padding_mask: [B, max_c] (True = 패딩된 가짜 채널)  │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ Model — Channel Mixer                                │
│                                                     │
│ Cross-Attention에서 padding_mask 적용:                │
│   패딩된 채널의 attention weight → -inf → softmax 후 0 │
│   → 가짜 채널이 Query의 정보 수집에 영향 안 줌            │
│                                                     │
│ 출력: [B, N, D] ← 채널 수와 완전히 무관                 │
└─────────────────────────────────────────────────────┘
```

## LitData 샘플 포맷

```python
{
    "eeg": np.array([C, 400]),       # 2초 윈도우 (float32)
    "coords": np.array([C, 3]),      # 전극 3D 좌표
    "channel_names": ["Fp1", ...],   # 디버깅/분석용
    "source_dataset": "tuab",        # 출처 추적
    "num_channels": C,               # 가변 채널 수 (int)
}
```

## 채널 Alias 처리

REVE Position Bank (543개 전극 커버)를 사용하여 매핑:
- 10-20, 10-10, 10-05 시스템
- EGI 256ch, BioSemi 128ch
- 대소문자 변형 (Cz/CZ, Fp1/FP1)

```
T3 → T7 (구→신 명명법 통일)
EEG1 → 매핑 불가 → 드롭
C3-A1 → 매핑 불가 → 드롭
```

## CAR 적용 순서 근거

| 방식 | 장점 | 단점 |
|---|---|---|
| 오프라인 적용 | 학습 시 연산 없음 | 원본 복구 불가, 다른 reference 실험 불가 |
| collate 후 on-the-fly | - | 패딩 채널이 평균에 포함되어 신호 왜곡 |
| **Dataset 내 on-the-fly** | 유연성 유지, 마스킹 불필요, 오버헤드 미미 | - |

**확정: Dataset 내 적용 (패딩 전)**

## 메모리 스파이크 대응 (필요 시)

REVE의 82%가 19~22ch이므로 대부분의 배치에서 패딩은 최소 수준.
128ch가 섞이는 경우는 드물고, 그때만 메모리가 일시적으로 증가.

만약 OOM 발생 시 (모델이 아닌 인프라 레벨에서 해결):
1. Gradient accumulation으로 실질 배치 크기 유지
2. 채널 수 기반 동적 batch_size (큰 채널 → 작은 배치)
3. 극단적으로 큰 채널(>128)만 예외적으로 제외

## 기각된 대안들

| 대안 | 기각 이유 |
|---|---|
| 채널 상한(64ch) + 서브샘플링 | "모든 채널 수 대응" 주장과 모순 |
| Bucket Sampling | 불필요한 복잡도, 82%가 저밀도라 버켓 불균형 |
| Mask-aware CAR | CAR을 패딩 전에 적용하면 불필요 |
